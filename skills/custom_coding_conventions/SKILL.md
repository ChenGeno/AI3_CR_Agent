# 团队自定义编程规范

## 描述

该 Skill 用于把团队内部约定的编程规范显式提供给 Code Review Agent，使模型在通用正确性、安全性、可维护性之外，还能按照团队自己的工程习惯给出审查意见。

## 使用场景

- 团队对异常处理、日志埋点、返回值结构有固定约束
- 团队禁止某些 API 或某类写法
- 团队要求关键模块必须满足统一的判空、防御式编程、配置读取规范

## 当前规范示例

1. 禁止使用 `except Exception` 直接吞掉所有异常，除非同时满足：
   - 明确记录日志
   - 返回值中保留必要错误上下文
   - 已说明为什么不能继续向上抛出

2. 禁止直接使用 `shell=True` 执行外部命令，除非：
   - 已做白名单校验
   - 无法改写为参数数组调用
   - 风险和原因已在代码注释中说明

3. 用户输入、外部系统返回值、配置中心数据一律视为不可信输入：
   - 进入核心逻辑前要做基础校验
   - 进入高风险能力前要做更严格校验

4. 对外返回的数据结构应保持稳定：
   - 不要静默删除既有字段
   - 不要随意修改字段语义
   - 新增错误字段时，需保证调用方可兼容

5. 对可能为空的对象访问必须显式防御：
   - 禁止依赖“通常不会为空”的经验假设
   - 对多级属性访问优先使用判空、默认值或安全包装函数

## 审查要求

- 当代码违反上述规范时，应明确指出对应的规范项
- 只有在有具体代码证据时才提出问题
- 不要把纯格式化或命名偏好包装成规范问题

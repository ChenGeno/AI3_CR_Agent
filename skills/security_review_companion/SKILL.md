# 安全风险专项审查

## 描述

该 Skill 用于增强 Code Review Agent 的安全专项审查能力。相比通用审查，它会更主动地关注输入边界、权限边界、外部命令执行、敏感信息暴露以及错误处理导致的安全后果。

## 重点关注

1. 命令执行风险
   - 是否把外部输入直接拼接进 shell 命令
   - 是否存在不必要的外部命令调用
   - 是否可以改成更安全的参数化执行方式

2. 输入校验缺失
   - 外部输入是否在进入关键逻辑前做校验
   - 是否把用户输入直接传给数据库、模板、命令、路径、配置解析等高风险接口

3. 敏感信息处理
   - 日志、错误返回、调试信息中是否泄露敏感字段
   - 异常栈或原始错误是否被直接暴露给调用方

4. 权限和边界
   - 是否绕过既有权限判断
   - 是否把原本内部使用的能力暴露给不可信调用方

5. 可被利用的失败路径
   - 广泛捕获异常后是否掩盖安全问题
   - 默认值或回退逻辑是否把失败转成危险的“成功”

## 审查要求

- 当存在安全风险时，优先指出风险链路，而不是只描述单行代码
- 尽量解释“输入从哪里来、为什么危险、可能造成什么后果”
- 如果风险能通过更小范围的改动缓解，应优先给出低侵入建议
- 没有充分证据时，不要夸大为安全漏洞
